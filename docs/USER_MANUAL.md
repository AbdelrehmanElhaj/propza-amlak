## 1. مقدمة النظام

### 1.1 نظرة عامة

**Propza** هو نظام متكامل لإدارة العقارات مبني على منصة Odoo 17، مُصمَّم خصيصاً للسوق السعودي. يغطي النظام دورة حياة العقار الكاملة:

- **إدارة المحفظة العقارية:** تسجيل العقارات وتتبع حالتها وربطها بالملاك.
- **دورة الإيجار الكاملة:** من استقبال طلب العميل إلى إبرام العقد وتحصيل الإيجار.
- **إدارة الصيانة:** طلبات الصيانة وأوامر العمل والعقود الدورية.
- **التكامل مع منصة إيجار الحكومية (ECRS):** رفع ومزامنة عقود الإيجار إلكترونياً.
- **الامتثال السعودي:** تجميد الإيجار بالرياض، الهوية الوطنية، IBAN، ضريبة القيمة المضافة.

### 1.2 متطلبات الوصول

| المتطلب | التفاصيل |
|---------|----------|
| **عنوان النظام** | `http://<عنوان-الخادم>:8069` |
| **المتصفح المدعوم** | Chrome 90+، Firefox 88+، Edge 90+، Safari 14+ |
| **الاتصال بالإنترنت** | مطلوب لاستخدام منصة إيجار |
| **الدقة الموصى بها** | 1280×720 أو أعلى |

> **ملاحظة:** النظام لا يدعم Internet Explorer بأي إصدار.

### 1.3 تسجيل الدخول

1. افتح المتصفح وانتقل إلى عنوان النظام.
2. أدخل **البريد الإلكتروني** في حقل "البريد الإلكتروني أو اسم المستخدم".
3. أدخل **كلمة المرور** في الحقل المخصص.
4. اضغط **تسجيل الدخول**.

**بيانات الدخول الافتراضية (للمسؤول):**
- البريد الإلكتروني: `admin@propza.sa`
- كلمة المرور: `admin`

> **تحذير أمني:** غيّر كلمة مرور المسؤول فور الانتهاء من الإعداد الأولي.

### 1.4 تسجيل الخروج

انقر على **اسم المستخدم** في أعلى يمين الشاشة، ثم اختر **تسجيل الخروج**.

## 2. التنقل في النظام

### 2.1 القوائم الرئيسية

يعرض النظام **ثماني قوائم رئيسية** في شريط التنقل العلوي (أو كأيقونات في الصفحة الرئيسية)، وتظهر حسب صلاحية كل دور:

| # | اسم القائمة | الوصف المختصر | الأدوار التي تراها |
|---|-------------|---------------|-------------------|
| 1 | **لوحة التحكم** | مؤشرات الأداء والرسوم البيانية | جميع الأدوار |
| 2 | **النظام الأساسي** | العقارات، الملاك، المستأجرون، الوسطاء، المعاينات | مدير + محاسب + موظف + مالك |
| 3 | **إدارة علاقات العملاء** | الطلبات، الجولات، الحجوزات | موظف + مدير |
| 4 | **إدارة عقود الإيجار** | العقود، الدفعات، العمولات، SADAD | مدير + محاسب + موظف |
| 5 | **إدارة الصيانة** | الطلبات، أوامر العمل، الفنيون | جميع الأدوار (مفلترة) |
| 6 | **منصة إيجار** | عقود ECRS، مكاتب الوساطة | مدير + محاسب + موظف |
| 7 | **المحاسبة المالية** | فواتير، مدفوعات، تقارير مالية | مدير + محاسب |
| 8 | **حسابي** | ملف المستخدم والوثائق | جميع الأدوار |

### 2.2 البحث والفلاتر

في كل قائمة توجد **شريط البحث** في أعلى الصفحة يدعم:

- **البحث النصي:** اكتب أي كلمة والنظام يبحث في الحقول المعرَّفة.
- **الفلاتر المسبقة (Filters):** اضغط السهم بجانب البحث → اختر من قائمة الفلاتر الجاهزة (مثال: "نشط"، "متأخر").
- **التجميع (Group By):** لتجميع النتائج حسب حقل معين (مثال: تجميع العقارات حسب النوع).
- **المفضلة (Favorites):** حفظ مجموعة فلاتر للاستخدام المتكرر.

### 2.3 طرق عرض القوائم

| طريقة العرض | الوصف | متى تستخدمها |
|-------------|-------|-------------|
| **قائمة (List)** | جدول بالسجلات | للاطلاع السريع والفرز |
| **كانبان (Kanban)** | بطاقات مجمَّعة حسب المرحلة | لتتبع سير العمل |
| **نموذج (Form)** | تفاصيل سجل واحد | للإنشاء والتعديل |
| **تحليل (Pivot)** | جدول محوري | للتقارير المقارَنة |
| **رسم بياني (Graph)** | مخططات بيانية | لتحليل الاتجاهات |

### 2.4 الاختصارات المفيدة

| الاختصار | الوظيفة |
|----------|---------|
| `Ctrl + K` أو `/` | فتح شريط البحث السريع |
| `Ctrl + S` | حفظ النموذج |
| `Escape` | إلغاء التعديل/إغلاق النافذة |
| زر **جديد** في شريط الأدوات | إنشاء سجل جديد |
| زر **حفظ يدوي** | حفظ السجل الحالي |

## 3. أدوار المستخدمين والصلاحيات

### 3.1 الأدوار السبعة

يعتمد النظام نظام صلاحيات متعدد المستويات يشمل **7 أدوار**:

#### أدوار الموظفين (هرمية تراكمية)

```
مدير النظام
  └── يشمل كل صلاحيات مدير العقارات +
مدير العقارات
  └── يشمل كل صلاحيات المحاسب +
محاسب
  └── يشمل كل صلاحيات موظف خدمة العملاء +
موظف خدمة العملاء
```

#### الأدوار الخاصة (مستقلة)

- **مالك عقار** — يرى عقاراته وعقوده ودفعاته فقط
- **فني صيانة** — يرى أوامر العمل المحددة له فقط
- **مستأجر (بوابة)** — وصول عبر البوابة الإلكترونية `/my` فقط

### 3.2 تفصيل صلاحيات كل دور

#### مدير النظام (group_pms_admin)

**الوصول:** صلاحيات كاملة بلا قيود.

يستطيع:
- إنشاء وتعديل وحذف أي سجل في النظام
- إدارة المستخدمين وتعيين الأدوار
- الوصول إلى إعدادات النظام وقوائم التطبيقات
- عرض جميع سجلات المزامنة مع منصة إيجار
- الوصول إلى **لوحة المالكين** الإدارية

---

#### مدير العقارات (group_pms_manager)

**الوصول:** إدارة كاملة للعمليات بدون إدارة المستخدمين.

يستطيع:
- إنشاء وتعديل وحذف العقارات والعقود والدفعات
- الموافقة على طلبات الصيانة وإنشاء أوامر العمل وعقود الصيانة الدورية
- عرض **لوحة المالكين** الإدارية
- إدارة **الإعدادات** في وحدات الصيانة والـ CRM

لا يستطيع:
- إدارة المستخدمين أو تعيين الأدوار (مخصص للمسؤول فقط)

---

#### محاسب (group_pms_accountant)

**الوصول:** الجانب المالي كاملاً + قراءة البيانات التشغيلية.

يستطيع:
- إنشاء وتعديل الدفعات والفواتير وعمولات الوسطاء
- عرض عقود الإيجار والعقارات (قراءة فقط)
- عرض تحليلات تكاليف الصيانة
- استخدام وحدة المحاسبة المالية كاملاً

لا يستطيع:
- تعديل العقارات أو العقود
- إدارة الصيانة (باستثناء التقارير المالية)

---

#### موظف خدمة العملاء (group_pms_agent)

**الوصول:** CRM كامل + تتبع العمليات اليومية.

يستطيع:
- إنشاء وإدارة طلبات CRM والجولات والحجوزات
- تقديم وتتبع طلبات الصيانة
- عرض العقارات والمستأجرين والملاك والوسطاء
- عرض جدول دفعات الإيجار (قراءة فقط)

لا يستطيع:
- إنشاء عقود إيجار (يحتاج مدير)
- الوصول إلى الدفعات والفواتير المالية التفصيلية

---

#### مالك عقار (group_pms_owner)

**الوصول:** محفظته الشخصية فقط — مفلتر بقواعد السجلات.

يرى فقط:
- العقارات المسجَّلة باسمه
- عقود الإيجار المرتبطة بعقاراته
- الدفعات الواردة على عقاراته
- طلبات ومعاينات الصيانة الخاصة بعقاراته

> **ملاحظة:** المالك مستخدم **داخلي** (يسجل الدخول للواجهة الخلفية)، وليس مستخدم بوابة.

---

#### فني صيانة (group_pms_technician)

**الوصول:** أوامر العمل المسندة إليه شخصياً.

يرى ويستطيع:
- عرض أوامر العمل المعينة له فقط
- تحديث حالة وتكاليف أوامر العمل
- عرض تفاصيل طلبات الصيانة المرتبطة بأوامر عمله

لا يستطيع:
- إنشاء طلبات صيانة جديدة
- عرض أوامر العمل لفنيين آخرين

---

#### مستأجر (بوابة) (group_pms_tenant_portal)

**الوصول:** بوابة الإنترنت `/my` فقط — لا وصول للواجهة الخلفية.

يرى ويستطيع من خلال البوابة:
- عقد الإيجار الخاص به (قراءة فقط)
- سجل الدفعات الخاص به (قراءة فقط)
- إنشاء وتتبع طلبات الصيانة لوحدته
- معاينات العقار المرتبطة بعقده (قراءة فقط)

للوصول إلى البوابة: `http://<عنوان-الخادم>/my`

### 3.4 قواعد السجلات (Record Rules)

قواعد السجلات تعمل على **مستوى قاعدة البيانات**، وليس فقط على مستوى الواجهة. هذا يعني:
- المالك والفني والمستأجر **لا يستطيعون الوصول** إلى بيانات الآخرين حتى عبر API.
- تسري هذه القواعد تلقائياً دون أي إعداد إضافي.
- للتحقق: سجّل الدخول بحساب مالك وستجد فقط عقاراته في جميع القوائم.

## 4. لوحة التحكم

### 4.1 الوصول

### 4.2 مؤشرات الأداء الرئيسية (KPIs)

## 5. النظام الأساسي

### 5.1 إدارة العقارات

**المسار:** النظام الأساسي ← العقارات

#### إضافة عقار جديد

1. انقر **جديد** في شريط الأدوات.
2. أدخل البيانات في التبويبات:

**تبويب الهوية:**

| الحقل | الوصف | ملاحظة |
|-------|-------|--------|
| **اسم العقار** | الاسم التجاري أو المرجعي | مطلوب |
| **اسم الوحدة** | رقم الشقة / الوحدة | مثال: "A-101" |
| **نوع العقار** | سكني / تجاري / صناعي / أرض | مطلوب |
| **رقم الصك** | رقم وثيقة الملكية | للامتثال القانوني |
| **رقم العنوان الوطني** | كود العنوان الوطني (مثال: RIYD0001) | 8 خانات |

**تبويب الموقع:**

| الحقل | الوصف |
|-------|-------|
| **المنطقة الإدارية** | من قائمة المناطق السعودية الـ 14 |
| **المدينة** | مرتبطة بالمنطقة |
| **الحي / الشارع** | العنوان التفصيلي |

**تبويب الأطراف:**

| الحقل | الوصف |
|-------|-------|
| **المالك** | اختر من قائمة الملاك المسجَّلين |
| **المستأجر الحالي** | يُملأ تلقائياً عند ربط عقد إيجار |

**تبويب التسعير:**

| الحقل | الوصف |
|-------|-------|
| **قيمة الإيجار السنوي** | بالريال السعودي |
| **سعر البيع** | بالريال السعودي |
| **مبلغ التأمين** | وديعة الضمان |

**تبويب المواصفات (موسَّع بعد تفعيل منصة إيجار):**

| الحقل | الوصف |
|-------|-------|
| **عدد الغرف** | للوحدات السكنية |
| **عدد دورات المياه** | |
| **المساحة (م²)** | المساحة الإجمالية |
| **مؤثَّث** | نعم / جزئياً / لا |
| **موقف سيارات** | نعم / لا |
| **مسبح** | نعم / لا |
| **حديقة** | نعم / لا |
| **مصعد** | نعم / لا |

3. انقر **حفظ**.

#### حالات العقار

| الحالة | الوصف | التحول التلقائي |
|--------|-------|----------------|
| **مسودة** | عقار غير مؤجَّر أو جديد | — |
| **مؤجَّر** | يوجد عقد إيجار ساري | عند تشغيل العقد |
| **مباع** | تم نقل الملكية | يدوياً |

### 5.2 إدارة الملاك

**المسار:** النظام الأساسي ← الملاك

الملاك هم جهات اتصال (`res.partner`) علاقتهم بالنظام محددة بعلامة **مالك عقار**.

#### إضافة مالك جديد

1. انقر **جديد**.
2. أدخل:
   - الاسم الكامل
   - رقم الجوال
   - البريد الإلكتروني
   - رقم الهوية الوطنية أو الإقامة
   - تاريخ انتهاء الهوية
   - رقم الآيبان (IBAN) — يجب أن يبدأ بـ SA ويكون 24 خانة
3. فعّل خيار **"مالك عقار"** في قسم الأدوار.
4. احفظ.

> **اختياري:** يمكن إنشاء حساب مستخدم للمالك بالضغط على **"إنشاء مستخدم"**، مما يتيح له الدخول للنظام ورؤية عقاراته فقط.

### 5.3 إدارة المستأجرين

**المسار:** النظام الأساسي ← المستأجرون

المستأجرون جهات اتصال بعلامة **مستأجر**.

#### إضافة مستأجر جديد

1. انقر **جديد**.
2. أدخل:
   - الاسم الكامل (بالعربي والإنجليزي)
   - رقم الجوال
   - البريد الإلكتروني
   - **نوع الهوية:** هوية وطنية / إقامة / هوية خليجي
   - **رقم الهوية** وتاريخ انتهائها
   - رقم العنوان الوطني
3. فعّل **"مستأجر"** في قسم الأدوار.
4. احفظ.

### 5.4 الوسطاء العقاريون

**المسار:** النظام الأساسي ← الوسطاء العقاريون

الوسطاء جهات اتصال بعلامة **وسيط عقاري**. تُستخدم سجلاتهم لربط عمولات الوساطة.

#### البيانات المطلوبة للوسيط

| الحقل | الوصف |
|-------|-------|
| **الاسم التجاري** | اسم مكتب الوساطة |
| **رقم الترخيص** | رقم ترخيص فال |
| **رقم السجل التجاري** | رقم الـ CR |
| **رقم ضريبة القيمة المضافة** | رقم الـ VAT (15 خانة) |
| **IBAN** | للتحويل البنكي للعمولات |

### 5.5 معاينات العقارات

**المسار:** النظام الأساسي ← معاينات العقارات

المعاينة هي تقرير حالة العقار عند **تسليمه أو استلامه** من المستأجر.

#### إنشاء معاينة جديدة

1. انقر **جديد**.
2. اختر:
   - **العقد المرتبط** (يملأ العقار تلقائياً)
   - **نوع المعاينة:** تسليم عند البداية / استلام عند النهاية / دورية
   - **تاريخ المعاينة**
   - **المفتِّش** (الموظف المسؤول)
3. في قسم **بنود المعاينة**، أضف غرفة أو مكوّناً في كل سطر:
   - اسم الغرفة / العنصر
   - الحالة: ممتازة / جيدة / مقبولة / رديئة / تالفة
   - ملاحظات
   - تكلفة الإصلاح (إن وجدت)
4. أدخل **الحالة العامة** للعقار.
5. احفظ.

> **استخدام عملي:** عند انتهاء عقد الإيجار، قارِن معاينة التسليم بمعاينة الاستلام لحساب الاستقطاعات من وديعة الضمان.

## 6. إدارة علاقات العملاء CRM

### 6.1 نظرة عامة على دورة CRM

```
طلب جديد → تأهيل كفرصة → حجز وحدة → تأكيد الحجز → تحويل لصفقة → إنشاء عقد إيجار
```

**المسار:** إدارة علاقات العملاء

### 6.2 الطلبات والفرص

**المسار:** إدارة علاقات العملاء ← الطلبات

#### إنشاء طلب جديد

1. انقر **جديد** في عرض الكانبان أو القائمة.
2. أدخل:

| الحقل | الوصف | مثال |
|-------|-------|------|
| **الموضوع** | وصف مختصر للطلب | "طلب شقة للإيجار - الرياض" |
| **العميل** | اختر أو أنشئ جهة اتصال | — |
| **نوع الطلب** | إيجار / شراء | — |
| **نوع العقار** | سكني / تجاري / أرض | — |
| **الميزانية** | من — إلى (بالريال) | 30,000 — 50,000 |
| **المنطقة المفضلة** | المنطقة الإدارية | الرياض |
| **عدد الغرف** | الحد الأدنى — الأقصى | 3 — 4 |
| **المساحة (م²)** | الحد الأدنى — الأقصى | 120 — 200 |
| **التأثيث** | مؤثَّث / غير مؤثَّث / أي | — |
| **المرافق** | موقف، مسبح، حديقة، مصعد | — |

3. احفظ. يظهر الطلب في لوحة الكانبان ضمن المرحلة الأولى.

#### تأهيل الطلب كفرصة

بعد الاتصال بالعميل والتحقق من جديته:
1. افتح سجل الطلب.
2. انقر زر **"تأهيل كفرصة"**.
3. تنتقل البطاقة للمرحلة التالية في لوحة الكانبان.

#### استخدام التوصيات الذكية (AI Match)

عند فتح فرصة، انقر **"توصيات ذكية"** ليعرض النظام أفضل العقارات المتاحة بناءً على:
- تطابق نوع العقار (+20 نقطة)
- تطابق المنطقة (+20 نقطة)
- تطابق عدد الغرف والمساحة (+10-20 نقطة)
- الميزانية والمرافق المطلوبة (+5-15 نقطة)

يُعرض حتى **8 عقارات** مقترحة مرتبةً تنازلياً حسب درجة التطابق.

#### لوحة الكانبان

تعرض الطلبات مجمَّعةً في **مراحل مخصصة** (قابلة للإضافة والتعديل من الإعدادات).

بطاقة كل طلب تعرض: اسم العميل، نوع الطلب، الميزانية، عدد الغرف، المساحة، عدد أيام الطلب المفتوح.

### 6.3 الجولات الميدانية

**المسار:** إدارة علاقات العملاء ← الجولات الميدانية

#### جدولة جولة ميدانية

1. انقر **جديد**.
2. اختر:
   - **الطلب/الفرصة** المرتبطة
   - **العقار** المراد زيارته
   - **تاريخ ووقت الجولة**
   - **الموظف المسؤول**
3. احفظ.

#### تسجيل نتيجة الجولة

بعد الجولة، افتح سجلها وحدث:

| الحقل | الخيارات |
|-------|----------|
| **النتيجة** | مجدولة / منجزة / ملغاة / لم يحضر العميل |
| **مستوى الاهتمام** | منخفض / متوسط / عالٍ |
| **ملاحظات العميل** | نص حر |

### 6.4 الحجوزات

**المسار:** إدارة علاقات العملاء ← الحجوزات

#### إنشاء حجز

من داخل سجل الفرصة:
1. انقر **"احجز العقار"**.
2. اختر العقار المراد حجزه.
3. حدد تاريخ بداية ونهاية الحجز.
4. احفظ.

#### تأكيد الحجز

افتح سجل الحجز وانقر **"تأكيد الحجز"** — تنتقل الحالة من "مسودة" إلى "مؤكد".

> **ملاحظة:** لا يمكن وجود أكثر من حجز مؤكد واحد لنفس العقار في نفس الوقت.

#### حالات الحجز

| الحالة | الوصف |
|--------|-------|
| **مسودة** | حجز غير مؤكد |
| **مؤكد** | حجز نشط — العقار محجوز للعميل |
| **منتهي** | انتهت مدة الحجز تلقائياً |
| **ملغي** | ألغي يدوياً |
| **محوَّل** | تم تحويله إلى عقد إيجار |

### 6.5 تحويل الصفقة إلى عقد إيجار

بعد تأكيد الحجز وموافقة العميل:

1. افتح سجل الفرصة.
2. انقر **"تحويل إلى صفقة"** (يظهر فقط بعد وجود حجز مؤكد).
3. تتحول الفرصة إلى **صفقة (Deal/Won)**.
4. انقر **"إنشاء عقد إيجار"**:
   - يُنشئ النظام عقد إيجار (`property.tenancy`) في حالة "مسودة".
   - يُنشئ تلقائياً عقد إيجار على منصة إيجار (`ejar.contract`) مرتبطاً به.
5. انتقل لوحدة **إدارة عقود الإيجار** لاستكمال بيانات العقد وتأكيده.

## 7. إدارة عقود الإيجار

### 7.1 عقود الإيجار

**المسار:** إدارة عقود الإيجار ← عقود الإيجار

#### دورة حياة العقد

```
مسودة → مؤكد (يُولَّد جدول الدفعات) → ساري (العقار يصبح "مؤجَّر") → منتهٍ
              ↑                                                         ↓
          (يمكن الإلغاء في أي وقت قبل الانتهاء)               (العقار يعود "شاغر")
```

| الحالة | الوصف | الإجراء التالي |
|--------|-------|---------------|
| **مسودة** | عقد منشأ لم يُؤكَّد | انقر "تأكيد" |
| **مؤكد** | عقد مؤكد وجدول الدفعات جاهز | انقر "بدء العقد" |
| **ساري** | عقد نشط والمستأجر في الوحدة | — |
| **منتهٍ** | انتهت مدة العقد | — |
| **ملغي** | أُلغي قبل انتهاء مدته | — |

#### إنشاء عقد إيجار جديد يدوياً

1. انقر **جديد**.
2. أدخل البيانات الأساسية:

| الحقل | الوصف | ملاحظة |
|-------|-------|--------|
| **العقار** | العقار المراد تأجيره | مطلوب |
| **المستأجر** | جهة اتصال المستأجر | مطلوب |
| **المالك** | يُملأ تلقائياً من العقار | |
| **تاريخ البداية** | تاريخ بدء العقد | مطلوب |
| **المدة** | بالأشهر أو السنوات | مطلوب |
| **تاريخ الانتهاء** | يُحسب تلقائياً | |
| **قيمة الإيجار** | الإيجار الدوري (ريال) | مطلوب |
| **دورية الدفع** | شهري / ربعي / نصف سنوي / سنوي | مطلوب |
| **طريقة الدفع** | تحويل بنكي / شيك / SADAD / نقداً | |
| **وديعة الضمان** | المبلغ المدفوع مقدماً | |

3. انقر **"تأكيد"**:
   - يُنشئ النظام جدول الدفعات تلقائياً حسب الدورية.
   - كل دفعة تأخذ تسلسلاً تلقائياً.
4. انقر **"بدء العقد"**:
   - تنتقل حالة العقد إلى "ساري".
   - تنتقل حالة العقار إلى "مؤجَّر".

> **تحذير — تجميد الإيجار بالرياض:** المنطقة التنظيمية (2025-2030) تمنع رفع إيجارات عقارات الرياض عند التجديد. النظام يُطبِّق هذا القيد تلقائياً عند محاولة تجاوزه.

#### إنهاء العقد

1. افتح سجل العقد الساري.
2. انقر **"إنهاء العقد"**.
3. تفتح **نافذة إنهاء العقد** للتأكيد واحتساب:
   - خصومات من وديعة الضمان (إن وجدت)
   - تسوية المبالغ المتبقية
4. أكد الإنهاء.

#### تجديد العقد

1. افتح العقد المنتهي أو الساري.
2. انقر **"تجديد العقد"**.
3. في نافذة التجديد:
   - حدد **تاريخ بداية جديد** ومدة التجديد.
   - أدخل **قيمة الإيجار الجديدة**.
   - إذا كان العقار في الرياض، تظهر **تحذير أصفر** بتجميد الإيجار.
4. أكد التجديد — يُنشئ النظام عقداً جديداً.

### 7.2 الدفعات

**المسار:** إدارة عقود الإيجار ← الدفعات

#### جميع الدفعات

يعرض **كل دفعات** جميع العقود. كل دفعة تحمل:

| الحقل | الوصف |
|-------|-------|
| **المرجع** | رقم تسلسلي تلقائي |
| **العقد** | العقد المرتبط |
| **العقار** | العقار المأجور |
| **المستأجر** | اسم المستأجر |
| **تاريخ الاستحقاق** | الموعد المحدد للدفع |
| **المبلغ** | قيمة الدفعة |
| **الحالة** | انظر الجدول أدناه |

| الحالة | الوصف | اللون في القائمة |
|--------|-------|-----------------|
| **معلقة** | لم يحن موعد الدفع | رمادي |
| **متأخرة** | تجاوز تاريخ الاستحقاق | أحمر |
| **مدفوعة** | تم التحصيل بالكامل | أخضر |
| **مدفوعة جزئياً** | تم التحصيل جزئياً | أصفر |
| **ملغاة** | دفعة ملغاة | رمادي فاتح |

> **ملاحظة:** يُحدِّث النظام تلقائياً كل دفعة "معلقة" تجاوز تاريخها إلى "متأخرة" عبر مهمة يومية مجدولة.

#### الدفعات المتأخرة

**المسار:** إدارة عقود الإيجار ← الدفعات ← دفعات متأخرة

قائمة مفلترة تعرض فقط الدفعات المتأخرة، مرتبةً حسب عدد أيام التأخير.

#### تسجيل دفعة

1. افتح سجل الدفعة المعلقة أو المتأخرة.
2. انقر **"تسجيل دفعة"**.
3. في النافذة المنبثقة:
   - **تاريخ الدفع:** تاريخ التحصيل الفعلي
   - **المبلغ المدفوع:** كامل أو جزئي
   - **طريقة الدفع:** تحويل بنكي / شيك / SADAD / نقداً
   - **ملاحظات:** أي ملاحظات إضافية
4. انقر **"تأكيد"** — تنتقل الدفعة إلى حالة "مدفوعة" أو "مدفوعة جزئياً".

#### إنشاء فاتورة

من سجل الدفعة، انقر **"إنشاء فاتورة"** لإنشاء فاتورة إيجار في وحدة المحاسبة.

### 7.3 عمولات الوسطاء

**المسار:** إدارة عقود الإيجار ← عمولات الوسطاء

> **ملاحظة:** هذه القائمة مرئية لـ **المحاسب والمدير** فقط.

#### إنشاء عمولة وساطة

1. انقر **جديد**.
2. اختر:
   - **الوسيط:** من قائمة الوسطاء
   - **عقد الإيجار:** العقد المرتبط (يملأ العقار والمستأجر تلقائياً)
   - **نوع العمولة:** نسبة مئوية / مبلغ ثابت
   - إذا اخترت **نسبة مئوية:** أدخل النسبة (% من الإيجار السنوي)
   - إذا اخترت **مبلغ ثابت:** أدخل المبلغ بالريال
   - **جدول الدفع:** دفعة واحدة عند التوقيع / شهري / نصفي (50% عند التوقيع + 50% بعد 6 أشهر)
3. احفظ — يُولِّد النظام جدول بنود الدفع.
4. انقر **"إنشاء فاتورة مورد"** لإرسال الفاتورة للمحاسبة.

#### جدول بنود الدفع

يعرض قسم **"بنود العمولة"** في نموذج العمولة جدولاً بـ:
- التسلسل
- تاريخ الاستحقاق
- المبلغ
- الحالة (معلقة / مدفوعة)

### 7.4 فواتير SADAD

**المسار:** إدارة عقود الإيجار ← فواتير SADAD

> **ملاحظة:** هذه القائمة مرئية لـ **المحاسب والمدير** فقط.
> **تنبيه:** الوحدة الحالية هي **محاكاة** لشبكة سداد، وليست ربطاً حقيقياً. الربط الفعلي يتطلب اتفاقية مع SAMA وشريك بنكي.

#### إنشاء فاتورة SADAD

1. انقر **جديد**.
2. اختر **دفعة الإيجار** المرتبطة (تملأ المبلغ تلقائياً).
3. حدد **تاريخ انتهاء الفاتورة**.
4. احفظ — يُنشئ النظام **رمز QR** تلقائياً.

#### رقم الفاتورة

يتكوّن رقم الفاتورة من 15 خانة:
- **4 خانات أولى:** رمز الجهة (biller code) — الافتراضي: 9999
- **11 خانة لاحقة:** مرجع العميل التلقائي

#### حالات الفاتورة

| الحالة | الوصف |
|--------|-------|
| **معلقة** | في انتظار الدفع |
| **مدفوعة** | تم الدفع (يدوياً أو عبر webhook) |
| **ملغاة** | ألغيت يدوياً |
| **منتهية الصلاحية** | تجاوزت تاريخ الانتهاء |

## 8. إدارة الصيانة

### 8.1 طلبات الصيانة

**المسار:** إدارة الصيانة ← طلبات الصيانة

#### دورة حياة الطلب

```
جديد → معتمد → مجدول → قيد التنفيذ → منجز
  ↕
ملغي (في أي وقت)
```

#### إنشاء طلب صيانة جديد

1. انقر **جديد**.
2. أدخل:

| الحقل | الوصف | ملاحظة |
|-------|-------|--------|
| **العقار** | العقار المعني | مطلوب |
| **عقد الإيجار** | العقد المرتبط (يملأ المستأجر تلقائياً) | |
| **التصنيف** | انظر الجدول أدناه | مطلوب |
| **الأولوية** | عادي / متوسط / عالٍ / عاجل | مطلوب |
| **الوصف** | تفاصيل المشكلة | |
| **المورد/المقاول** | إذا كان العمل خارجياً | |

**تصنيفات الصيانة:**

| التصنيف | الأمثلة |
|---------|---------|
| **سباكة** | تسريب، انسداد، إصلاح حنفيات |
| **كهرباء** | أعطال، إضاءة، توصيلات |
| **تكييف** | صيانة مكيفات، تنظيف فلاتر |
| **دهانات** | طلاء جدران، إزالة بقع |
| **نجارة** | أبواب، نوافذ، خزائن |
| **تنظيف** | تنظيف عام، غسيل |
| **مكافحة حشرات** | رش ومعالجة |
| **أجهزة كهربائية** | ثلاجة، غسالة، فرن |
| **أخرى** | أي خدمة أخرى |

3. انقر **"موافقة"** (للمديرين والموظفين) لتحريك الطلب من "جديد" إلى "معتمد".

#### متابعة سير الطلب

| الزر | الانتقال | من يستطيعه |
|------|----------|------------|
| **موافقة** | جديد → معتمد | موظف+ |
| **جدولة** | معتمد → مجدول | موظف+ |
| **بدء العمل** | مجدول → قيد التنفيذ | موظف + فني |
| **إتمام** | قيد التنفيذ → منجز | موظف + فني |
| **إلغاء** | أي حالة → ملغي | موظف+ |

#### الطلبات العاجلة

**المسار:** إدارة الصيانة ← عاجلة

قائمة مفلترة تعرض فقط الطلبات بأولوية **"عاجل" (الدرجة 3)**، مرتبة حسب تاريخ الإنشاء.

### 8.2 أوامر العمل

**المسار:** إدارة الصيانة ← أوامر العمل

أمر العمل هو تفويض مفصَّل لفني محدد بناءً على طلب صيانة.

#### إنشاء أمر عمل

1. من سجل طلب الصيانة في حالة "معتمد"، انقر **"إنشاء أمر عمل"**.
2. أو انتقل لقائمة أوامر العمل وانقر **جديد**.
3. أدخل:

| الحقل | الوصف |
|-------|-------|
| **الطلب المرتبط** | ربط بطلب الصيانة |
| **الفني** | اختر الفني المسؤول |
| **تاريخ ووقت الجدولة** | موعد التنفيذ |
| **تكلفة المواد** | ريال |
| **تكلفة العمالة** | ريال |
| **تكلفة النقل** | ريال |
| **التكلفة الإجمالية** | تُحسب تلقائياً |
| **مَن يتحمل التكلفة** | المالك / المستأجر / مُقسَّمة |

4. احفظ — يتلقى الفني إشعاراً بأمر العمل.

> **ملاحظة:** الفني يرى أمر العمل هذا فقط عند تسجيل دخوله — قواعد السجلات تمنع رؤية أوامر الفنيين الآخرين.

### 8.3 عقود الصيانة الدورية

**المسار:** إدارة الصيانة ← عقود الصيانة الدورية

عقد الصيانة الدورية ينشئ طلبات صيانة تلقائياً في المواعيد المحددة.

#### إنشاء عقد دوري

1. انقر **جديد**.
2. أدخل:

| الحقل | الوصف |
|-------|-------|
| **المورد/المقاول** | شركة الصيانة |
| **العقارات** | قائمة العقارات المشمولة |
| **التصنيف** | نوع الصيانة الدورية |
| **التكرار** | شهري / ربع سنوي / نصف سنوي / سنوي |
| **تاريخ البداية** | بداية سريان العقد |
| **آخر خدمة** | يُحدَّث تلقائياً |
| **الخدمة القادمة** | يُحسب تلقائياً |

3. احفظ.

> النظام ينشئ طلبات الصيانة تلقائياً يومياً عند حلول موعد كل خدمة.

### 8.4 المقاولون والفنيون

**المسار:** إدارة الصيانة ← المقاولون والفنيون

قائمة جهات الاتصال المعلَّمة كـ **"فني صيانة"**. يمكن:

- إضافة فنيين وتحديد **تخصصاتهم** (من قائمة التخصصات في الإعدادات).
- عرض أوامر العمل المرتبطة بكل فني.
- إنشاء حساب مستخدم للفني للدخول ومتابعة أوامر عمله.

## 9. منصة إيجار

### 9.1 نظرة عامة

**منصة إيجار** هي المنصة الحكومية السعودية لتسجيل عقود الإيجار إلكترونياً (نظام ECRS v4). يتكامل Propza مع المنصة بشكل **غير متزامن** عبر قائمة انتظار وظيفية، مما يضمن استجابة الواجهة دون انتظار.

**المسار:** منصة إيجار

### 9.2 ملف مكتب الوساطة

قبل إرسال أي عقد، يجب إعداد **ملف مكتب الوساطة** الذي يمثل هوية شركتك على المنصة.

**المسار:** منصة إيجار ← ملفات مكاتب الوساطة

1. انقر **جديد**.
2. أدخل:

| الحقل | الوصف |
|-------|-------|
| **اسم المكتب (عربي)** | كما هو في السجل التجاري |
| **اسم المكتب (إنجليزي)** | — |
| **رقم السجل التجاري** | CR Number |
| **رقم ضريبة القيمة المضافة** | VAT Number (15 خانة) |
| **رقم ترخيص فال** | رقم ترخيص وسيط عقاري |
| **ممثل الشركة** | جهة الاتصال المسؤولة |
| **الشركة** | الكيان القانوني في Odoo |

3. احفظ.

### 9.3 إنشاء عقد إيجار على منصة إيجار

**المسار:** منصة إيجار ← عقود إيجار

#### دورة حياة العقد (9 حالات)

```
مسودة
  ↓ [بدء الإعداد]
جارٍ الإعداد
  ↓ [إضافة أطراف + وحدات → تأكيد الاكتمال]
جاهز للإرسال
  ↓ [إرسال إلى إيجار]
جارٍ الإرسال (معالجة في الخلفية)
  ↓
بانتظار الموافقة
  ↓
موافق عليه                    أو               مرفوض
  ↓                                               ↓
منتهي / ملغي                              يمكن التعديل وإعادة الإرسال
```

#### إنشاء عقد جديد

> **ملاحظة:** عادةً يُنشأ عقد إيجار المنصة تلقائياً عند تحويل صفقة CRM. لكن يمكن إنشاؤه يدوياً كالتالي:

1. انقر **جديد**.
2. في تبويب **"البيانات الأساسية"**:

| الحقل | الوصف |
|-------|-------|
| **ملف مكتب الوساطة** | اختر الملف المُعدَّ مسبقاً |
| **عقد الإيجار** | ربط بـ property.tenancy |
| **نوع العقد** | إيجار جديد / تجديد / نقل ملكية |
| **نوع الاستخدام** | سكني / تجاري / صناعي |
| **تاريخ البداية والنهاية** | مدة العقد |
| **قيمة الإيجار السنوي** | بالريال السعودي |
| **جدول الدفع** | شهري / ربعي / نصف سنوي / سنوي |
| **رسم الوساطة** | لا تتجاوز 2.5% من الإيجار السنوي |

3. انقر **"بدء الإعداد"** — تنتقل الحالة إلى "جارٍ الإعداد".

#### إضافة أطراف العقد

في تبويب **"الأطراف"**، أضف كل طرف:

| الحقل | الوصف |
|-------|-------|
| **الدور** | مُؤجِّر / مستأجر / ممثل |
| **نوع الجهة** | فرد / شركة |
| **جهة الاتصال** | اختر من السجلات |
| **رقم الهوية** | هوية وطنية / إقامة / سجل تجاري |
| **رقم الجوال** | للتحقق من المنصة |

#### إضافة الوحدات

في تبويب **"الوحدات"**، أضف العقار المستأجَر:

| الحقل | الوصف |
|-------|-------|
| **العقار** | اختر من سجلات العقارات |
| **نوع الوحدة** | شقة / فيلا / مكتب / مستودع... |
| **حالة التأثيث** | مؤثَّث / غير مؤثَّث |
| **عدد الغرف** | — |

#### إرسال العقد

1. انقر **"تأكيد الاكتمال"** — تنتقل الحالة إلى "جاهز للإرسال".
2. انقر **"إرسال إلى إيجار"**:
   - يُضيف النظام الطلب لقائمة الانتظار.
   - تنتقل الحالة إلى "جارٍ الإرسال".
   - معالج في الخلفية يرسل البيانات لـ API منصة إيجار.
3. انتظر وصول نتيجة المعالجة عبر **webhook** من المنصة.

#### في حالة الرفض

إذا رفضت المنصة العقد:
1. افتح سجل العقد وانظر **سبب الرفض** في سجلات المزامنة.
2. صحّح البيانات المطلوبة.
3. أعد الإرسال.

### 9.4 سجلات المزامنة

**المسار:** منصة إيجار ← سجلات المزامنة

> **ملاحظة:** هذه القائمة مرئية لـ **مسؤول النظام** فقط.

تعرض سجل شامل لكل استدعاء API مع منصة إيجار:
- اتجاه الاتصال (صادر / وارد)
- الطريقة (POST / GET / PATCH)
- المسار (endpoint)
- الحالة (نجاح / خطأ / إعادة محاولة)
- الرسالة التفصيلية للخطأ

## 10. المحاسبة المالية

**المسار:** المحاسبة المالية

هذه الوحدة هي وحدة المحاسبة المدمجة في Odoo 17، مُوسَّعة بمتطلبات الامتثال السعودي (ZATCA).

### 10.1 الفواتير

**المسار:** المحاسبة المالية ← العملاء ← فواتير العملاء

تُنشأ فواتير الإيجار تلقائياً عند تسجيل الدفعات أو يدوياً من سجلات الدفعات.

**أنواع الفواتير في السياق العقاري:**
- فواتير إيجار (من سجلات الدفعات)
- فواتير عمولات الوسطاء (من وحدة العمولات)
- فواتير الصيانة (من وحدة الصيانة)

### 10.2 المدفوعات

**المسار:** المحاسبة المالية ← العملاء ← مدفوعات العملاء

لتسجيل مدفوعات العملاء وتسويتها مع الفواتير المفتوحة.

### 10.3 التقارير المالية

| التقرير | الوصف |
|---------|-------|
| **الأستاذ العام** | كل الحركات المالية مفصَّلة |
| **ميزان المراجعة** | ملخص الأرصدة |
| **تقرير ضريبة القيمة المضافة** | للتقديم الدوري لـ ZATCA |
| **بيان الإيرادات والمصروفات** | الأرباح والخسائر |
| **الميزانية العمومية** | الأصول والخصوم |

## 11. الإعدادات والإدارة

> **ملاحظة:** الوصول لهذا القسم مقتصر على **مدير النظام**.

### 11.1 إدارة المستخدمين

**المسار:** الإعدادات ← المستخدمون وشركات → المستخدمون

#### إضافة مستخدم جديد

1. انقر **جديد**.
2. أدخل:
   - **الاسم الكامل**
   - **البريد الإلكتروني** (يُستخدم للدخول)
3. في قسم **الصلاحيات ← إدارة العقارات**، اختر الدور المناسب:
   - مدير النظام
   - مدير العقارات
   - محاسب
   - موظف خدمة عملاء
   - مالك عقار
   - فني صيانة
4. انقر **"حفظ"** ثم **"إرسال دعوة"** — يتلقى المستخدم بريداً لضبط كلمة المرور.

#### تعديل دور مستخدم موجود

1. افتح سجل المستخدم.
2. عدّل الدور في قسم الصلاحيات.
3. احفظ — تُطبَّق الصلاحيات فوراً.

#### إيقاف حساب مستخدم

1. افتح سجل المستخدم.
2. من القائمة العلوية، انقر **"إجراء" ← أرشفة**.
3. يُوقَف الحساب دون حذف سجلاته.

### 11.2 إدارة المناطق الإدارية

**المسار:** منصة إيجار ← الإعدادات ← المناطق الإدارية السعودية

يحتوي النظام على المناطق الـ 14 المسجَّلة رسمياً في المملكة العربية السعودية مع مدنها. لإضافة مدينة جديدة أو تعديل بيانات منطقة:

1. افتح سجل المنطقة.
2. عدّل القائمة الفرعية للمدن.
3. احفظ.

### 11.3 إعدادات منصة إيجار

**المسار:** منصة إيجار ← الإعدادات

| الإعداد | الوصف |
|---------|-------|
| **رابط API** | عنوان API منصة إيجار |
| **المفتاح السري** | للتحقق من صحة الـ webhooks (HMAC-SHA256) |
| **مدة انتهاء الفاتورة** | عدد الأيام لصلاحية فواتير SADAD |
| **رمز الجهة (SADAD)** | رمز الجهة في شبكة سداد |

### 11.4 إعدادات CRM

**المسار:** إدارة علاقات العملاء ← الإعدادات ← المراحل

إدارة مراحل خط سير CRM (Kanban pipeline):
1. افتح أي مرحلة لتعديل اسمها أو ترتيبها.
2. انقر **جديد** لإضافة مرحلة جديدة.
3. خصص **نسبة الاحتمال** لكل مرحلة.

## ملحق ب: الأسئلة الشائعة

**س: هل يمكن لمستأجر واحد أن يمتلك أكثر من عقد إيجار نشط؟**
ج: نعم، يمكن للمستأجر امتلاك عقود متعددة على عقارات مختلفة.

**س: ماذا يحدث لجدول الدفعات عند إلغاء العقد؟**
ج: الدفعات المعلقة تُلغى تلقائياً، والدفعات المدفوعة تبقى كسجل مالي.

**س: هل يمكن تعديل قيمة دفعة بعد إنشائها؟**
ج: الدفعات تُنشأ من جدول العقد. لتعديل المبلغ، يجب التواصل مع المحاسب لإنشاء تسوية.

**س: هل يرسل النظام إشعارات للمستأجرين قبل موعد الدفع؟**
ج: هذه الميزة قيد التطوير. حالياً يُتابع المحاسب والموظفون دفعات المتأخرين يدوياً أو من لوحة التحكم.

**س: ماذا يعني "جارٍ الإرسال" في منصة إيجار؟**
ج: يعني أن الطلب في قائمة الانتظار وسيُرسَل قريباً. المعالجة غير متزامنة وقد تستغرق دقيقة.

**س: كيف أمنح مالكاً وصولاً للنظام؟**
ج: من سجل الشريك (المالك)، انقر "إنشاء مستخدم". سيُنشئ النظام حساباً بصلاحيات "مالك عقار" تلقائياً.
